# Family Fabricator (Fam|Fab) — AI-Powered Revit Family Generator

FamFab is an open-source Revit add-in that generates parametric families from images. Snap one or more photos of a piece of furniture, a light fixture, or any building component, and FamFab uses a vision AI model — Claude, Gemini, or Kimi — to create a fully parametric Revit family — complete with 3D geometry, parameters, and subcategories.

## How It Works

1. **Open the Family Editor** with the appropriate template (Generic Model, Furniture, etc.)
2. **Click the FamFab button** on the ribbon (ArchSmarter tab)
3. **Pick one or more images** — photos, catalog screenshots, or sketches (e.g. front, side, top, detail views)
4. **FamFab sends the images to your chosen model provider** with a structured skill prompt and receives a JSON family definition
5. **Review the 3D preview** — an interactive Three.js viewer lets you rotate, zoom, and adjust parameters in real time
6. **Click Generate** — the JSON is executed in the Family Editor to create the geometry, parameters, reference planes, and subcategories

You can also **load a JSON file directly** to skip the API call, or use the **Refine** feature to iterate on the design with follow-up prompts.

## Key Features

- **Image to family** — Describe or photograph what you want and get a parametric Revit family
- **Interactive 3D preview** — Review and tweak parameters before committing to Revit
- **Bring Your Own Key (BYOK)** — Uses your own Anthropic API key. No subscription, no account creation, no data sent anywhere except the Claude API. Your key is stored locally in `%AppData%`
- **Editable skill prompt** — The prompt that tells Claude how to generate families is embedded in the add-in but fully editable in the source. Fork it, tune it, make it yours
- **Supports Revit 2020-2027** — Multi-target build covers .NET Framework 4.8 (2020-2024) and .NET 8/10 (2025-2027)
- **Geometry types** — Extrusions, sweeps, and blends with parametric expressions
- **JSON schema** — A documented schema for family definitions that can be generated by any LLM or written by hand

## Requirements

- Revit 2025 or later
- An [Anthropic API key](https://console.anthropic.com/)
- WebView2 Runtime (included with Windows 11; available for Windows 10)

## Installation

### From source

1. Clone this repository
2. Install the [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0) (required for Revit 2025+)
3. Open `ArchSmarterFamFab.sln` in Visual Studio
4. Select the build configuration for your Revit version (e.g., `Debug R25`)
5. Build the solution — the post-build step copies the add-in to your Revit Addins folder automatically

### First run

1. Open Revit and go to the **ArchSmarter** tab
2. Click the **Settings** button in the FamFab panel
3. Enter your Anthropic API key
4. Select a model (Claude Sonnet is recommended for speed/quality balance)

## Usage

1. Open a new family using an appropriate template
2. Click the **FamFab** button
3. Browse to one or more images of the object you want to create (e.g. front, side, top, detail views)
4. Optionally enter a family name and additional context (e.g., "36 inch wide, wall-mounted")
5. Click **Generate** and wait for the API response
6. In the preview window, review the 3D model and adjust parameters using the side panel
7. Click **Generate Family** to create the geometry in the Family Editor
8. Add dimensions and constraints as needed to complete the family

### Loading JSON directly

Click **Load JSON** in the generate window to load a previously saved family definition. This is useful for testing, sharing family definitions, or working with JSON generated outside the add-in.

## BYOK — Bring Your Own Key

FamFab does not include an API key, proxy, or backend service. You provide your own Anthropic API key, which is stored locally in `%AppData%\ArchSmarter\FamFab\FamFab.json`. Image and prompt data is sent directly to the Anthropic API — nowhere else.

API costs depend on model choice and image size. A typical family generation call costs a few cents.

## Contributing

Contributions are welcome. Some areas where help would be especially useful:

- **Skill prompt improvements** — Better prompts produce better families. The prompt is in `EmbeddedContent/SKILL.md`
- **New geometry types** — The executor currently supports extrusions, sweeps, and blends. Revolves and swept blends would expand what families can be generated
- **Family schema extensions** — Support for connectors, nested families, or additional parameter types

## Future Improvements

- **Multi-image input** — Allow multiple reference images (front, side, top, detail) for more accurate geometry
- **Parametric dimensions** — Automatically create labeled dimension strings so geometry is driven by parameter values
